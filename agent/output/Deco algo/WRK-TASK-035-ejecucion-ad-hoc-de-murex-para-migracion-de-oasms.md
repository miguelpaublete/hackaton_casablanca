# Ejecución ad hoc de Murex para migración de OASms

## Objective
Discutir y coordinar la ejecución ad hoc de Murex con el equipo de reporting para desbloquear la migración de OASms de Mentor a Smart Data.

## Implementation Notes
La migración está bloqueada por falta de información de Murex. Se adjuntó un Excel con las OASms a migrar y se necesita identificar cuáles están en uso para evitar migrar reportes vacíos. La ejecución ad hoc se realizará en la weekly de sensibilidades del miércoles.

## Acceptance Criteria
- [ ] Se realiza la ejecución ad hoc de Murex.
- [ ] Se identifican las OASms en uso.
- [ ] Se desbloquea la migración de OASms a Smart Data.

## Test Plan
Verificar que la información de Murex esté disponible en Smart Data y que las OASms migradas sean las correctas según el inventario.
