# Ejecución ad hoc de Murex para desbloquear migración de OASms

## Objective - what must be achieved
Realizar una ejecución ad hoc de Murex el próximo fin de semana para verificar si la información necesaria para la migración de OASms está disponible y así desbloquear el proceso de migración a Smart Data.

## Implementation Notes - technical hints from the conversation
La ejecución debe coordinarse con el equipo de reporting en la reunión semanal de sensibilidades del miércoles. Es necesario identificar si la información de Murex está llegando correctamente a Smart Data o si se requiere alguna acción adicional.

## Acceptance Criteria - testable bullets
- [ ] La ejecución ad hoc de Murex se realiza en el plazo acordado.
- [ ] Se verifica la disponibilidad de la información requerida para la migración de OASms.
- [ ] Se documentan los resultados y se comunica al equipo de migración.

## Test Plan - how to verify
Revisar los logs y reportes generados por la ejecución ad hoc de Murex y confirmar con el equipo de migración si el bloqueo ha sido resuelto.
